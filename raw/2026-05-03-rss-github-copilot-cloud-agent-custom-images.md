[Copilot cloud agent](https://docs.github.com/copilot/concepts/agents/coding-agent/about-coding-agent) now starts up over 20% faster, thanks to optimized runner environments built with [GitHub Actions custom images](https://docs.github.com/actions/using-github-hosted-runners/using-larger-runners/managing-larger-runners#configuring-a-custom-image).

When you assign an issue to Copilot, start a task from the **Agents** tab, or mention `@copilot` in a pull request, the agent spins up a cloud-based environment to do its work. By prebuilding that environment with a custom Actions image, startup overhead has been significantly reduced, getting Copilot to work on your code faster than before.

This builds on the [50% startup improvement shipped in March](https://github.blog/changelog/2026-03-19-copilot-coding-agent-now-starts-work-50-percent-faster/), continuing to shorten the feedback loop when you work with Copilot cloud agent.

To learn more, see the [Copilot cloud agent documentation](https://docs.github.com/copilot/concepts/agents/coding-agent/about-coding-agent).
